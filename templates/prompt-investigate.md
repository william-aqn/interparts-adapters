## Что нужно решить

1. **Есть ли у сайта JSON API, пригодный для реверса?**
   Критерии: JSON-ответы на запросы (желательно POST/GET к поиску/каталогу),
   стабильные URL, внятные поля. Одиночные JSON-запросы к аналитике/трекингу
   не считаются.

2. **Какой режим адаптера оптимален?**
   - `api` — если нашёлся явный JSON-эндпоинт поиска/каталога, который можно
     дёргать без Playwright.
   - `http` — если нужной API нет, но HTML каталога статичен и парсится Cheerio.
   - `browser` — если сайт SPA, контент рисуется JS, нужна Playwright-страница.

3. **Если режим `api` — какие эндпоинты использовать?**
   Для каждого кандидата укажи URL-шаблон, метод, пример тела запроса и
   ожидаемый ключ в ответе.

4. **Если режим `http` / `browser` — где каталог и какие селекторы подсказать?**
   Укажи предполагаемый URL поисковой/каталожной страницы (можно относительный),
   и, если видно из HTML, — селекторы для ключевых элементов: контейнер карточки,
   наименование, цена, бренд, наличие, артикул.

5. **Нужна ли авторизация** для доступа к каталогу?

6. **Подсказка `siteId`** — короткий slug из хоста (`a-z 0-9 _ -`, 2–40 символов).

7. **Подсказка `siteName`** — человекочитаемое имя (на основе title и домена).

## Формат ответа

Верни РОВНО один JSON-блок (без пояснительного текста вокруг) с такой схемой:

```json title="analysis.json"
{
  "hasApi": true,
  "proposedMode": "api",
  "reasoning": "Краткое объяснение выбора — 2-4 предложения на русском",
  "suggestedSiteId": "example-ru",
  "suggestedName": "Example Parts",
  "needsAuth": false,
  "apiCandidates": [
    {
      "url": "https://example.com/api/search",
      "method": "POST",
      "confidence": "high",
      "requestExample": "{\"query\":\"1K0615301AA\",\"limit\":20}",
      "responseKey": "results",
      "notes": "Основной поисковый эндпоинт; возвращает JSON с массивом results"
    }
  ],
  "catalogUrl": null,
  "selectorHints": {},
  "knownIssues": null,
  "apiNotes": "POST /api/search с query partNumber; требует header X-CSRF-Token",
  "customInstructions": "При разборе ответа: price — строка с пробелами как разделителями тысяч",
  "suggestedHealthCheckQuery": "1K0615301AA"
}
```

Правила:
- `proposedMode` ∈ `"api" | "http" | "browser"`.
- Для `proposedMode = "api"` — `apiCandidates` должно быть НЕ пустым и отсортировано по убыванию `confidence`.
- Для `proposedMode = "http" | "browser"` — `apiCandidates` пустой массив `[]`, но нужно заполнить `catalogUrl` и, по возможности, `selectorHints` (ключи: `resultContainer`, `partNumber`, `name`, `price`, `brand`, `availability`).
- `confidence` ∈ `"high" | "medium" | "low"`.
- `selectorHints` — плоский объект `{ ключ: "CSS-selector" }`. Не используй XPath.
- Все строковые поля могут быть `null`, если достоверной информации нет.
- Никакого текста вокруг JSON. Никаких комментариев внутри JSON.
